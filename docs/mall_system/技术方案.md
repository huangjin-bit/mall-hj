## 三、核心功能实现

### 3.1 认证流程（Authentication）

#### 3.1.1 登录流程

```
1. 用户提交用户名和密码
   ↓
2. 后端验证用户名和密码
   ↓
3. 查询用户的角色和权限信息
   ↓
4. 生成 JWT Token（包含 user_id、username、roles）
   ↓
5. 将 Token 存储到 Redis（key: login_token:user_id, value: Token, TTL: 2小时）
   ↓
6. 返回 Token 给前端
   ↓
7. 前端后续请求携带 Token（Header: Authorization: Bearer {token}）
```

#### 3.1.2 Token 结构

```json
{
  "user_id": 1,
  "username": "admin",
  "nickname": "超级管理员",
  "dept_id": 101,
  "roles": ["admin"],
  "exp": 1704931200,
  "iat": 1704924800
}
```

#### 3.1.3 代码实现

**登录接口**：

```java
@PostMapping("/login")
public Result<LoginVO> login(@RequestBody LoginDTO loginDTO) {
    // 1. 验证验证码
    validateCaptcha(loginDTO.getUuid(), loginDTO.getCaptcha());

    // 2. 验证用户名和密码
    SysUser user = userService.getByUsername(loginDTO.getUsername());
    if (user == null) {
        throw new BusinessException("用户不存在");
    }
    if (!BCrypt.checkpw(loginDTO.getPassword(), user.getPassword())) {
        throw new BusinessException("密码错误");
    }
    if (user.getStatus() == 0) {
        throw new BusinessException("账号已停用");
    }

    // 3. 查询用户角色和权限
    Set<String> roles = roleService.getRolesByUserId(user.getId());
    Set<String> perms = menuService.getPermsByUserId(user.getId());

    // 4. 生成 Token
    String token = JwtUtils.createToken(user.getId(), user.getUsername(), roles);

    // 5. 存储 Token 到 Redis
    redisTemplate.opsForValue().set(
        "login_token:" + user.getId(),
        token,
        2,
        TimeUnit.HOURS
    );

    // 6. 记录登录日志
    operLogService.recordLogin(user.getId(), loginDTO.getIp());

    // 7. 返回登录信息
    LoginVO loginVO = new LoginVO();
    loginVO.setToken(token);
    loginVO.setUser(user);
    loginVO.setRoles(roles);
    loginVO.setPerms(perms);
    return Result.success(loginVO);
}
```

**JWT 工具类**：

```java
public class JwtUtils {
    private static final String SECRET = "mall_system_secret_key_2026";
    private static final long EXPIRATION = 7200; // 2小时

    public static String createToken(Long userId, String username, Set<String> roles) {
        Date now = new Date();
        Date expiryDate = new Date(now.getTime() + EXPIRATION * 1000);

        return Jwts.builder()
            .subject(String.valueOf(userId))
            .claim("username", username)
            .claim("roles", roles)
            .issuedAt(now)
            .expiration(expiryDate)
            .signWith(Keys.hmacShaKeyFor(SECRET.getBytes()))
            .compact();
    }

    public static Claims parseToken(String token) {
        return Jwts.parser()
            .verifyWith(Keys.hmacShaKeyFor(SECRET.getBytes()))
            .build()
            .parseSignedClaims(token)
            .getPayload();
    }

    public static Long getUserId(String token) {
        Claims claims = parseToken(token);
        return Long.parseLong(claims.getSubject());
    }
}
```

### 3.2 授权流程（Authorization）

#### 3.2.1 权限验证流程

```
1. 用户请求接口（携带 Token）
   ↓
2. 网关验证 Token 有效性
   ↓
3. 解析 Token 获取用户 ID 和角色
   ↓
4. 查询用户的权限列表（从 Redis 缓存获取，缓存不存在则查询数据库）
   ↓
5. 检查接口所需的权限标识（注解定义）
   ↓
6. 验证用户是否拥有该权限
   ↓
7. 通过：放行请求 | 不通过：返回 403 Forbidden
```

#### 3.2.2 权限注解

```java
@Target({ElementType.METHOD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
public @interface RequiresPermissions {
    String value(); // 权限标识，如 "product:category:add"
}
```

#### 3.2.3 AOP 权限拦截

```java
@Aspect
@Component
public class PermissionAspect {
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    @Around("@annotation(requiresPermissions)")
    public Object checkPermission(ProceedingJoinPoint joinPoint, RequiresPermissions requiresPermissions) throws Throwable {
        // 1. 获取当前登录用户
        Long userId = SecurityUtils.getUserId();
        if (userId == null) {
            throw new BusinessException("未登录");
        }

        // 2. 从 Redis 获取用户权限
        String cacheKey = "user_perms:" + userId;
        Set<String> userPerms = (Set<String>) redisTemplate.opsForValue().get(cacheKey);

        if (userPerms == null) {
            // 缓存不存在，查询数据库
            userPerms = menuService.getPermsByUserId(userId);
            redisTemplate.opsForValue().set(cacheKey, userPerms, 1, TimeUnit.HOURS);
        }

        // 3. 检查权限
        String requiredPerm = requiresPermissions.value();
        if (!userPerms.contains(requiredPerm)) {
            throw new BusinessException("无权限访问");
        }

        // 4. 执行目标方法
        return joinPoint.proceed();
    }
}
```

### 3.3 菜单树构建

#### 3.3.1 菜单树算法

```java
public List<MenuTree> buildMenuTree(List<SysMenu> allMenus, Long parentId) {
    return allMenus.stream()
        .filter(menu -> menu.getParentId().equals(parentId))
        .map(menu -> {
            MenuTree node = new MenuTree();
            node.setId(menu.getId());
            node.setName(menu.getName());
            node.setPath(menu.getPath());
            node.setComponent(menu.getComponent());
            node.setIcon(menu.getIcon());
            node.setChildren(buildMenuTree(allMenus, menu.getId()));
            return node;
        })
        .collect(Collectors.toList());
}
```

#### 3.3.2 用户菜单查询

```java
@Service
public class MenuService {
    @Autowired
    private SysMenuMapper menuMapper;

    @Autowired
    private SysRoleMenuMapper roleMenuMapper;

    @Autowired
    private SysUserRoleMapper userRoleMapper;

    public List<MenuTree> getUserMenus(Long userId) {
        // 1. 查询用户的角色
        List<Long> roleIds = userRoleMapper.getRoleIdsByUserId(userId);

        // 2. 查询角色的菜单 ID
        List<Long> menuIds = roleMenuMapper.getMenuIdsByRoleIds(roleIds);

        // 3. 查询菜单详情
        List<SysMenu> menus = menuMapper.selectBatchIds(menuIds);

        // 4. 过滤按钮类型（只返回目录和菜单）
        List<SysMenu> menuTree = menus.stream()
            .filter(m -> m.getMenuType() == 0 || m.getMenuType() == 1)
            .collect(Collectors.toList());

        // 5. 构建菜单树
        return buildMenuTree(menuTree, 0L);
    }
}
```

### 3.4 数据权限实现

#### 3.4.1 数据权限枚举

```java
public enum DataScope {
    ALL(1, "全部数据"),
    CUSTOM(2, "自定义数据"),
    DEPT(3, "本部门数据"),
    DEPT_AND_CHILD(4, "本部门及以下数据"),
    SELF(5, "仅本人数据");

    private final Integer code;
    private final String desc;
}
```

#### 3.4.2 数据权限 AOP

```java
@Aspect
@Component
public class DataScopeAspect {
    @Autowired
    private SysDeptService deptService;

    @Before("@annotation(dataScope)")
    public void doBefore(JoinPoint point, DataScope dataScope) {
        // 1. 获取当前用户
        SysUser user = SecurityUtils.getLoginUser();

        // 2. 获取用户角色的数据权限
        Integer scope = user.getDataScope();

        // 3. 构建数据权限 SQL
        StringBuilder sql = new StringBuilder();
        if (scope == DataScope.ALL.getCode()) {
            // 全部数据，不添加条件
            return;
        } else if (scope == DataScope.CUSTOM.getCode()) {
            // 自定义数据
            sql.append(" AND dept_id IN (").append(user.getDeptIds()).append(")");
        } else if (scope == DataScope.DEPT.getCode()) {
            // 本部门数据
            sql.append(" AND dept_id = ").append(user.getDeptId());
        } else if (scope == DataScope.DEPT_AND_CHILD.getCode()) {
            // 本部门及以下数据
            List<Long> deptIds = deptService.getDeptAndChildIds(user.getDeptId());
            sql.append(" AND dept_id IN (").append(deptIds).append(")");
        } else if (scope == DataScope.SELF.getCode()) {
            // 仅本人数据
            sql.append(" AND create_by = ").append(user.getId());
        }

        // 4. 将 SQL 条件设置到 ThreadLocal
        DataScopeContext.setSql(sql.toString());
    }
}
```

#### 3.4.3 MyBatis-Plus 数据权限插件

```java
@Component
public class DataScopeInterceptor implements InnerInterceptor {
    @Override
    public void beforeQuery(Executor executor, MappedStatement ms, Object parameter, RowBounds rowBounds, ResultHandler resultHandler, BoundSql boundSql) throws SQLException {
        // 获取数据权限 SQL
        String dataScopeSql = DataScopeContext.getSql();
        if (StringUtils.isNotBlank(dataScopeSql)) {
            // 拼接到原始 SQL
            String originalSql = boundSql.getSql();
            String newSql = originalSql + dataScopeSql;
            // 使用反射修改 SQL
            ReflectionUtils.setFieldValue(boundSql, "sql", newSql);
        }
    }
}
```

### 3.5 操作日志记录

#### 3.5.1 操作日志注解

```java
@Target({ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
public @interface Log {
    String title();           // 模块标题
    OperType operType();      // 操作类型
    boolean saveRequestData() default true;   // 是否保存请求参数
    boolean saveResponseData() default true;  // 是否保存响应参数
}
```

#### 3.5.2 操作日志 AOP

```java
@Aspect
@Component
public class LogAspect {
    @Autowired
    private SysOperLogService operLogService;

    @Around("@annotation(log)")
    public Object around(ProceedingJoinPoint point, Log log) throws Throwable {
        long startTime = System.currentTimeMillis();
        Object result = null;
        Exception exception = null;

        try {
            // 执行目标方法
            result = point.proceed();
            return result;
        } catch (Exception e) {
            exception = e;
            throw e;
        } finally {
            // 记录操作日志
            recordOperLog(point, log, result, exception, startTime);
        }
    }

    private void recordOperLog(ProceedingJoinPoint point, Log log, Object result, Exception exception, long startTime) {
        // 1. 获取当前用户
        SysUser currentUser = SecurityUtils.getLoginUser();

        // 2. 获取请求信息
        HttpServletRequest request = ServletUtils.getRequest();
        String requestUrl = request.getRequestURI();
        String requestMethod = request.getMethod();

        // 3. 获取方法信息
        MethodSignature signature = (MethodSignature) point.getSignature();
        String method = signature.getDeclaringType().getName() + "." + signature.getName();

        // 4. 获取请求参数
        String requestParams = null;
        if (log.saveRequestData()) {
            Object[] args = point.getArgs();
            requestParams = JSON.toJSONString(args);
        }

        // 5. 获取返回结果
        String responseResult = null;
        if (log.saveResponseData() && result != null) {
            responseResult = JSON.toJSONString(result);
        }

        // 6. 计算耗时
        long costTime = System.currentTimeMillis() - startTime;

        // 7. 构建操作日志对象
        SysOperLog operLog = new SysOperLog();
        operLog.setTitle(log.title());
        operLog.setOperType(log.operType().getCode());
        operLog.setMethod(method);
        operLog.setRequestUrl(requestUrl);
        operLog.setRequestMethod(requestMethod);
        operLog.setOperParam(requestParams);
        operLog.setOperResult(responseResult);
        operLog.setOperUserId(currentUser.getId());
        operLog.setOperUserName(currentUser.getUsername());
        operLog.setOperIp(IpUtils.getIpAddr(request));
        operLog.setStatus(exception == null ? 1 : 0);
        operLog.setErrorMsg(exception != null ? exception.getMessage() : null);
        operLog.setCostTime(costTime);
        operLog.setCreateTime(new Date());

        // 8. 异步保存日志
        operLogService.asyncSave(operLog);
    }
}
```

## 五、安全设计

### 5.1 密码安全

#### 5.1.1 密码加密
- **算法**：BCrypt
- **强度**：10（默认）
- **示例**：
  ```
  原始密码：admin123
  加密后：$2a$10$7JB720yubVSZvUI0rEqK/.VqGOZTH.ulu33dHOiBE8ByOhJIrdAu2
  ```

#### 5.1.2 密码策略
- 最小长度：8 位
- 复杂度要求：必须包含字母和数字
- 禁止使用常见密码（如：123456、password）
- 定期强制修改密码（90天）

#### 5.1.3 密码重置流程
1. 管理员在后台重置用户密码
2. 系统生成随机密码
3. 通过邮件或短信发送给用户
4. 用户首次登录强制修改密码

### 5.2 接口安全

#### 5.2.1 Token 管理
- **Token 有效期**：2 小时
- **刷新 Token**：在 Token 过期前 30 分钟可刷新
- **Token 存储**：Redis（key: login_token:user_id）
- **Token 失效**：登出时删除 Redis 中的 Token

#### 5.2.2 接口限流
- **策略**：基于 IP + 用户 ID 的限流
- **阈值**：普通用户 100 次/分钟，VIP 用户 300 次/分钟
- **实现**：Redis + Lua 脚本

#### 5.2.3 SQL 注入防护
- **MyBatis 参数绑定**：所有参数使用 `#{}` 占位符
- **参数校验**：使用 `@Validated` 注解校验参数
- **特殊字符过滤**：过滤单引号、双引号、分号等特殊字符

#### 5.2.4 XSS 防护
- **输入过滤**：使用 `Jsoup.clean()` 过滤 HTML 标签
- **输出转义**：使用 `EscapeUtils.escape()` 转义特殊字符
- **CSP 策略**：设置 Content-Security-Policy 响应头

### 5.3 敏感数据保护

#### 5.3.1 数据脱敏
- **手机号**：保留前 3 位和后 4 位，中间用 * 代替（如：158****8888）
- **邮箱**：保留前 2 位和域名，中间用 * 代替（如：ad***@163.com）
- **身份证号**：保留前 6 位和后 4 位，中间用 * 代替

#### 5.3.2 日志脱敏
- **密码**：日志中不记录密码字段
- **敏感字段**：手机号、邮箱、身份证号自动脱敏
- **实现**：使用 Logback 的替换转换器

```xml
<configuration>
    <conversionRule conversionWord="mask" converterClass="com.mall.common.log.MaskingConverter" />
```

```java
        // 检查数据库连接

            DataSource dataSource = SpringUtils.getBean(DataSource.class);

        } catch (Exception e) {

        }

        return health;
    }
}
```

## 十、常见问题

### 10.1 Token 过期如何处理？

**答**：Token 过期后，前端需要跳转到登录页面重新登录。如果需要实现自动刷新 Token，可以在 Token 中包含刷新时间，在过期前自动调用刷新接口。

### 10.2 如何防止用户重复登录？

**答**：使用 Redis 存储用户的登录 Token，每次登录时检查是否已存在 Token，如果存在则删除旧 Token，确保同一用户只能有一个有效登录会话。

### 10.3 如何实现动态权限？

**答**：权限信息存储在数据库中，用户登录时从数据库加载权限，并通过 AOP 拦截器验证权限。修改权限后，清除 Redis 缓存，下次请求时会重新加载权限。

### 10.4 如何实现跨模块权限控制？

**答**：通过网关统一验证 Token 和权限，网关将用户信息（user_id、roles、perms）放入请求头，传递给下游微服务，下游微服务从请求头中获取用户信息，进行本地权限验证。
