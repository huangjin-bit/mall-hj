# 商城系统 RBAC 权限管理系统 - 需求文档

## 一、项目概述

### 1.1 项目背景
商城系统权限管理模块（mall_system）是基于 RBAC（Role-Based Access Control，基于角色的访问控制）模型设计的后台管理系统权限控制核心。该系统为商城管理后台提供用户认证、授权、菜单管理、操作审计等基础功能。

### 1.2 项目目标
- 实现灵活的权限管理，支持细粒度的功能权限控制
- 提供完整的用户、角色、菜单、部门管理体系
- 记录用户操作日志，满足审计要求
- 为商城各业务模块提供统一的权限控制基础

### 1.3 适用范围
- 商城管理后台（mall-admin）
- 各微服务模块的权限控制
- 系统管理员、商品管理员、订单管理员等角色的权限分配

## 二、功能需求

### 2.1 用户管理（sys_user）

#### 2.1.1 用户基本信息
- **用户名**：登录账号，全局唯一
- **昵称**：显示名称
- **密码**：BCrypt 加密存储
- **邮箱**：用户联系邮箱
- **手机号**：联系电话
- **性别**：0-未知，1-男，2-女
- **头像**：头像 URL 地址
- **状态**：0-停用，1-正常

#### 2.1.2 组织关系
- **所属部门**：关联 sys_dept 表
- **角色关联**：通过 sys_user_role 关联多个角色

#### 2.1.3 登录信息
- **最后登录 IP**：记录最后一次登录的 IP 地址
- **最后登录时间**：记录最后一次登录的时间戳

#### 2.1.4 审计字段
- 创建人、创建时间
- 更新人、更新时间
- 备注

#### 2.1.5 功能需求
1. **用户查询**：支持按用户名、部门、状态等条件查询
2. **用户新增**：创建新用户，分配部门和角色
3. **用户修改**：修改用户基本信息、部门、角色
4. **用户删除**：删除用户（逻辑删除）
5. **重置密码**：管理员重置用户密码
6. **状态控制**：启用/停用用户

---

### 2.2 角色管理（sys_role）

#### 2.2.1 角色基本信息
- **角色名称**：角色显示名称
- **角色权限字符串**：角色标识（如：admin、product_admin）
- **排序号**：角色显示顺序
- **状态**：0-停用，1-正常
- **备注**：角色说明

#### 2.2.2 数据权限（data_scope）
- **1-全部数据**：可查看所有数据
- **2-自定义数据**：可查看指定部门的数据
- **3-本部门数据**：只能查看本部门数据
- **4-本部门及以下数据**：可查看本部门及子部门数据
- **5-仅本人数据**：只能查看自己创建的数据

#### 2.2.3 功能需求
1. **角色查询**：查询角色列表
2. **角色新增**：创建新角色，分配菜单权限
3. **角色修改**：修改角色信息、菜单权限、数据权限
4. **角色删除**：删除角色
5. **权限分配**：为角色分配菜单权限

---

### 2.3 菜单管理（sys_menu）

#### 2.3.1 菜单类型
- **0-目录**：一级菜单（如：商品管理、订单管理）
- **1-菜单**：二级菜单（如：分类管理、品牌管理）
- **2-按钮**：功能按钮权限（如：查询、新增、修改、删除）

#### 2.3.2 菜单属性
- **菜单名称**：显示名称
- **路由地址**：前端路由路径
- **组件路径**：Vue 组件路径
- **重定向地址**：重定向路由
- **权限标识**：后端接口权限标识（如：product:category:list）
- **菜单图标**：前端显示图标
- **显示顺序**：同级菜单的排序
- **显示状态**：0-隐藏，1-显示
- **菜单状态**：0-停用，1-正常

#### 2.3.3 菜单树结构
```
系统管理（目录）
  └── 用户管理（菜单）
        ├── 用户查询（按钮）
        ├── 用户新增（按钮）
        ├── 用户修改（按钮）
        ├── 用户删除（按钮）
        └── 重置密码（按钮）
```

#### 2.3.4 功能需求
1. **菜单查询**：查询菜单树
2. **菜单新增**：新增菜单（目录/菜单/按钮）
3. **菜单修改**：修改菜单信息
4. **菜单删除**：删除菜单

---

### 2.4 部门管理（sys_dept）

#### 2.4.1 部门属性
- **部门名称**：部门显示名称
- **父部门 ID**：0 表示顶级部门
- **祖级列表**：从根到当前部门的祖先 ID 链（如：0,100,101）
- **排序号**：同级部门的排序
- **负责人**：部门负责人姓名
- **联系电话**：部门联系电话
- **邮箱**：部门联系邮箱
- **状态**：0-停用，1-正常

#### 2.4.2 功能需求
1. **部门查询**：查询部门树
2. **部门新增**：新增部门
3. **部门修改**：修改部门信息
4. **部门删除**：删除部门

---

### 2.5 用户角色关联（sys_user_role）

#### 2.5.1 多对多关系
一个用户可以拥有多个角色，一个角色可以分配给多个用户。

#### 2.5.2 功能需求
1. 为用户分配角色
2. 撤销用户角色
3. 查询用户的角色列表
4. 查询角色的用户列表

---

### 2.6 角色菜单关联（sys_role_menu）

#### 2.6.1 多对多关系
一个角色可以拥有多个菜单权限，一个菜单可以分配给多个角色。

#### 2.6.2 功能需求
1. 为角色分配菜单权限
2. 撤销角色的菜单权限
3. 查询角色的菜单权限列表
4. 查询菜单可分配的角色列表

---

### 2.7 操作日志（sys_oper_log）

#### 2.7.1 日志属性
- **模块标题**：操作的模块名称
- **操作类型**：0-其他，1-新增，2-修改，3-删除
- **方法名称**：调用的 Java 方法名
- **请求 URL**：请求的接口地址
- **请求方式**：GET/POST/PUT/DELETE
- **请求参数**：接口请求参数（JSON）
- **返回参数**：接口返回结果（JSON）
- **操作员 ID**：操作用户的 ID
- **操作员名称**：操作用户的用户名
- **操作 IP**：操作用户的 IP 地址
- **操作状态**：0-失败，1-成功
- **错误消息**：操作失败时的错误信息
- **消耗时间**：接口执行耗时（毫秒）
- **操作时间**：操作发生的时间戳

#### 2.7.2 功能需求
1. 自动记录用户的增删改操作
2. 查询操作日志（支持时间范围、用户、模块等条件）
3. 日志详情查看（请求参数、返回结果）
4. 日志导出

## 三、权限控制需求

### 3.1 预置角色

#### 3.1.1 超级管理员（admin）
- **权限范围**：所有功能模块
- **数据权限**：全部数据
- **菜单权限**：所有菜单

#### 3.1.2 商品管理员（product_admin）
- **权限范围**：商品管理模块
- **数据权限**：全部数据
- **菜单权限**：
  - 分类管理（查询、新增、修改、删除）
  - 品牌管理（查询、新增、修改、删除）
  - 属性分组（查询、新增、修改、删除）
  - 属性管理（查询、新增、修改、删除）
  - SPU 管理（查询、新增、修改、删除、上架）
  - SKU 管理（查询、新增、修改、删除）
  - 审核日志（查询）

#### 3.1.3 订单管理员（order_admin）
- **权限范围**：订单管理模块
- **数据权限**：全部数据
- **菜单权限**：
  - 订单列表（查询、修改）
  - 支付记录（查询）
  - 退货申请（查询、处理）
  - 退货原因（查询、新增、修改、删除）
  - 订单设置（修改）

### 3.2 权限控制流程

```
用户登录 → 获取用户角色 → 获取角色菜单权限 → 构建用户菜单树
          ↓
          用户请求 → 验证 Token → 获取用户权限 → 验证接口权限 → 允许/拒绝
```

### 3.3 菜单模块划分

| 模块编号 | 模块名称 | 路由路径 | 图标 |
|---------|---------|---------|-----|
| 1000 | 商品管理 | /product | shopping |
| 2000 | 会员管理 | /member | user |
| 3000 | 订单管理 | /order | list |
| 4000 | 营销管理 | /marketing | price-tag |
| 5000 | 仓储管理 | /ware | box |
| 6000 | 系统管理 | /system | setting |

## 四、非功能需求

### 4.1 安全需求
1. **密码加密**：使用 BCrypt 算法加密存储密码
2. **SQL 注入防护**：使用 MyBatis 参数绑定防止 SQL 注入
3. **XSS 防护**：对用户输入进行转义处理
4. **敏感信息保护**：操作日志中不记录密码等敏感信息
5. **会话管理**：基于 JWT Token 的无状态会话管理

### 4.2 性能需求
1. **登录响应时间**：< 500ms
2. **菜单查询响应时间**：< 300ms
3. **权限验证响应时间**：< 50ms
4. **操作日志写入**：异步写入，不阻塞业务流程

### 4.3 可扩展性需求
1. **多租户支持**：预留租户 ID 字段，支持未来 SaaS 化
2. **国际化支持**：菜单名称支持多语言
3. **动态权限**：支持运行时动态调整角色权限

### 4.4 可维护性需求
1. **代码规范**：遵循阿里巴巴 Java 开发手册
2. **注释完整**：所有公共方法添加 JavaDoc 注释
3. **日志记录**：关键操作记录日志，包含上下文信息
4. **异常处理**：统一异常处理，返回友好的错误信息

## 五、数据约束

### 5.1 唯一性约束
- sys_user.username：用户名全局唯一
- sys_role.role_key：角色权限字符串唯一

### 5.2 必填约束
- sys_user：username、nickname、password 必填
- sys_role：role_name、role_key 必填
- sys_menu：name、menu_type 必填
- sys_dept：dept_name 必填

### 5.3 关联约束
- 删除用户时，级联删除 sys_user_role 中的记录
- 删除角色时，级联删除 sys_user_role 和 sys_role_menu 中的记录
- 删除菜单时，级联删除子菜单和 sys_role_menu 中的记录
- 删除部门时，需先处理子部门和用户关联
