# mall_product 商品库 — 需求文档

## 1. 模块概述

`mall_product` 是 Mall-HJ 项目的商品服务数据库，服务于 `mall-product` 微服务。

核心职责：
- 三级分类管理（无限层级树形结构，实际使用三级）
- 品牌管理，品牌与分类多对多关联
- 属性体系：属性分组 → 属性定义 → 预定义可选值
- SPU/SKU 商品管理
- 商品审核流程
- 商品上下架管理

---

## 2. 功能需求

### 2.1 分类管理

| 功能 | 描述 |
|---|---|
| 三级分类 | 如：手机数码 → 手机通讯 → 手机 |
| 分类属性 | 三级分类绑定属性组，不同分类有不同的规格模板 |
| 分类品牌 | 三级分类关联品牌（多对多），发商品时只选关联的品牌 |

### 2.2 品牌管理

| 功能 | 描述 |
|---|---|
| 品牌信息 | 名称、Logo、介绍 |
| 品牌关联 | 一个品牌可属于多个分类（如华为→手机+电脑+手表） |

### 2.3 属性体系（B2B2C 模式）

管理员定义属性模板，商家发商品时从中选择：

| 层级 | 说明 | 操作者 |
|---|---|---|
| 属性分组 | 如"主体""屏幕""网络" | 管理员 |
| 属性定义 | 如"颜色""内存""CPU型号" | 管理员 |
| 预定义可选值 | 如"黑色""128GB""A16" | 管理员 |

- 基本属性：描述商品是什么（CPU、屏幕、电池），所有SKU共享
- 销售属性：区分SKU的属性（颜色、存储），决定用户买哪个

### 2.4 SPU/SKU 管理

| 概念 | 说明 |
|---|---|
| SPU | 标准商品单元，如"iPhone 15"，共享基本属性 |
| SKU | 库存单元，如"iPhone 15 黑色 128GB"，有独立价格和库存 |

商家发布商品流程：
1. 选择三级分类 → 系统加载属性模板和关联品牌
2. 填写基本信息（名称、品牌、简介）
3. 填写基本属性（CPU、屏幕等）
4. 勾选销售属性（颜色、存储）→ 系统自动生成SKU组合
5. 为每个SKU设置价格、库存、图片
6. 上传商品图片和富文本详情
7. 提交审核

### 2.5 商品审核

| 功能 | 描述 |
|---|---|
| 提交审核 | 商家提交后进入待审核状态 |
| 审核通过 | 管理员审核通过后可上架 |
| 审核拒绝 | 记录拒绝原因，通知商家修改 |
| 审核日志 | 记录每次审核操作（谁审核、结果、原因） |

### 2.6 商品上下架

| 功能 | 描述 |
|---|---|
| 上架 | 审核通过的商品可上架销售 |
| 下架 | 可随时下架，不影响已生成的订单 |
| 新品标记 | 标记为新品，前端展示"新品"标签 |
| 推荐标记 | 标记为推荐商品，首页推荐位展示 |

## 3. 数据库表清单

| # | 表名 | 类型 | 说明 |
|---|---|---|---|
| 1 | pms_category | 配置表 | 三级分类 |
| 2 | pms_brand | 配置表 | 品牌 |
| 3 | pms_category_brand_relation | 关联表 | 分类-品牌多对多 |
| 4 | pms_attr_group | 配置表 | 属性分组（绑定三级分类） |
| 5 | pms_attr | 配置表 | 属性定义 |
| 6 | pms_attr_value | 配置表 | 属性预定义可选值（管理员维护） |
| 7 | pms_attr_attrgroup_relation | 关联表 | 属性-分组关联 |
| 8 | pms_spu_info | 业务表 | SPU基本信息 |
| 9 | pms_spu_info_desc | 业务表 | SPU富文本描述 |
| 10 | pms_spu_images | 业务表 | SPU图片集 |
| 11 | pms_spu_attr_value | 业务表 | SPU基本属性值 |
| 12 | pms_sku_info | 业务表 | SKU信息（价格、库存） |
| 13 | pms_sku_images | 业务表 | SKU图片 |
| 14 | pms_sku_sale_attr_value | 业务表 | SKU销售属性值 |
| 15 | pms_spu_audit_log | 日志表 | 商品审核日志 |

## 4. 数据关系

```
pms_category（三级分类）
    │
    ├── pms_attr_group（属性组）
    │       └── pms_attr（属性定义）via pms_attr_attrgroup_relation
    │               └── pms_attr_value（预定义可选值）
    │
    ├── pms_brand via pms_category_brand_relation（多对多）
    │
    └── pms_spu_info（SPU）
            ├── pms_spu_info_desc（富文本）
            ├── pms_spu_images（图片集）
            ├── pms_spu_attr_value（基本属性值）
            ├── pms_spu_audit_log（审核日志）
            │
            └── pms_sku_info（SKU）
                    ├── pms_sku_images（SKU图片）
                    └── pms_sku_sale_attr_value（销售属性值）
```

## 5. 三表对比：attr_value vs spu_attr_value vs sku_sale_attr_value

| | pms_attr_value | pms_spu_attr_value | pms_sku_sale_attr_value |
|---|---|---|---|
| **是什么** | 可选值模板 | 商品基本参数 | 规格组合选项 |
| **谁维护** | 管理员 | 商家（从模板选/手填） | 商家（从模板选） |
| **数据量** | 全局一份 | 每个SPU一份 | 每个SKU一份 |
| **示例** | 颜色可选：黑/白/蓝 | iPhone15的CPU=A16 | 这个SKU：颜色=黑+存储=128G |
| **前端位置** | 后台管理 | 商品详情"规格参数" | 商品详情"选择颜色/存储" |
