## 2. 核心概念

### 2.1 SPU vs SKU

```
SPU（标准商品单元）= 商品种类
  → 描述"这是个什么商品"
  → 基本属性共享（CPU、屏幕、电池）
  → 一个 SPU 对应多个 SKU

SKU（库存单元）= 具体可购买的最小单位
  → 描述"用户买的是哪一个"
  → 销售属性不同（颜色、存储）
  → 有独立的价格、库存、图片
  → 用户下单、秒杀、购物车操作的都是 SKU

示例：
  SPU: iPhone 15
  ├── SKU-1: 黑色 128GB  ¥5999
  ├── SKU-2: 黑色 256GB  ¥6999
  ├── SKU-3: 白色 128GB  ¥5999
  └── SKU-4: 白色 256GB  ¥6999
```

### 2.2 属性三层体系

```
第1层：pms_attr_value（可选值模板，管理员维护）
  颜色可选值：黑色、白色、蓝色

第2层：pms_spu_attr_value（商品基本参数，商家填写/选择）
  iPhone 15 的 CPU = A16（从模板选或手填）

第3层：pms_sku_sale_attr_value（规格组合，商家选择后自动生成）
  SKU-1 的 颜色=黑色 + 存储=128GB（从模板选）
```

## 3. 商家发布商品流程

```
┌─ 第1步：选分类 ────────────────────────────────────────────┐
│  商家选择三级分类 → 系统自动加载：                            │
│  - 该分类的属性组 + 属性 + 可选值（从 pms_attr_value）        │
│  - 该分类关联的品牌列表（从 pms_category_brand_relation）     │
└────────────────────────────────────────────────────────────┘
            │
            ▼
┌─ 第2步：填基本信息 ────────────────────────────────────────┐
│  商品名称、品牌（下拉选关联品牌）、简介、重量                  │
└────────────────────────────────────────────────────────────┘
            │
            ▼
┌─ 第3步：填基本属性 ────────────────────────────────────────┐
│  有预定义值的属性 → 渲染下拉框，从 pms_attr_value 选          │
│  无预定义值的属性 → 渲染输入框，手填                          │
│  结果写入 pms_spu_attr_value                                │
└────────────────────────────────────────────────────────────┘
            │
            ▼
┌─ 第4步：选销售属性，生成 SKU ──────────────────────────────┐
│  勾选销售属性的可选值（颜色：黑/白，存储：128G/256G）          │
│  系统自动生成 2×2=4 个 SKU 组合                              │
│  商家为每个 SKU 填：价格、库存、图片                          │
│  结果写入 pms_sku_info + pms_sku_sale_attr_value            │
└────────────────────────────────────────────────────────────┘
            │
            ▼
┌─ 第5步：上传图片和详情 ────────────────────────────────────┐
│  SPU 级图片 → pms_spu_images                                │
│  富文本详情 → pms_spu_info_desc                              │
│  SKU 级图片 → pms_sku_images                                │
└────────────────────────────────────────────────────────────┘
            │
            ▼
┌─ 第6步：提交审核 ──────────────────────────────────────────┐
│  audit_status=0（待审核）                                    │
│  管理员审核：通过(1) / 拒绝(2)                                │
│  审核记录写入 pms_spu_audit_log                              │
│  通过后商家可手动上架（publish_status=1）                     │
└────────────────────────────────────────────────────────────┘
```

## 4. 后端校验规则

### 4.1 属性值校验

```java
// 商家提交属性值时的校验逻辑
if (attr.getValueType() != 0) {
    // 有预定义值的属性 → 必须从 pms_attr_value 中选
    boolean valid = attrValueMapper.exists(
        attr_id = attr.getId() AND value = submittedValue
    );
    if (!valid) throw new BizException("属性值不在可选范围内");
} else {
    // 手动输入的属性 → 直接保存
}
```

### 4.2 SKU 唯一性

```sql
-- 同一 SPU 下，销售属性组合不能重复
-- 通过查询 pms_sku_sale_attr_value 判断
-- 如：已有 SKU (颜色=黑色,存储=128GB)，不能再创建相同组合
```

## 6. 索引设计说明

| 表 | 索引 | 用途 |
|---|---|---|
| pms_category | idx_parent_id | 查子分类 |
| pms_category_brand_relation | uk_category_brand | 防重复关联 |
| pms_attr_group | idx_category_id | 查分类下的属性组 |
| pms_attr_value | idx_attr_id | 查属性的可选值 |
| pms_attr_attrgroup_relation | uk_attr_group | 防重复关联 |
| pms_spu_info | idx_category_id, idx_brand_id | 按分类/品牌筛选商品 |
| pms_spu_info | idx_audit_status | 审核列表查询 |
| pms_spu_info | idx_create_by | 商家查自己的商品 |
| pms_sku_info | idx_spu_id | 查SPU下的所有SKU |
| pms_sku_info | idx_price | 价格排序/筛选 |
| pms_sku_sale_attr_value | idx_spu_id | 查SPU有哪些销售属性可选 |
