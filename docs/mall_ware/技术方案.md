# mall_ware 仓储库 — 技术方案

## 1. 技术选型

| 项目 | 选型 |
|---|---|
| 数据库 | MySQL 8.0+, InnoDB, utf8mb4 |
| ID 生成 | 雪花算法（MyBatis-Plus ASSIGN_ID） |
| 分布式事务 | Seata 2.6.0（AT 模式，与 mall-order 联动） |
| 库存预警 | 定时任务扫描 low_stock 阈值 |

---

## 2. 库存锁定/解锁流程（Seata 分布式事务）

```
┌─ Seata 全局事务 ──────────────────────────────────────────┐
│                                                            │
│  mall-order（事务发起方）                                   │
│    │                                                       │
│    ├─ 1. 创建订单（oms_order + oms_order_item）            │
│    │                                                       │
│    ├─ 2. Feign 调用 mall-ware 锁定库存                     │
│    │     │                                                 │
│    │     ▼                                                 │
│    │   mall-ware（事务参与者）                               │
│    │     │                                                 │
│    │     ├─ 创建工作单（wms_ware_order_task）              │
│    │     │   task_status = 0（待锁定）                     │
│    │     │                                                 │
│    │     ├─ 创建工作单详情（wms_ware_order_task_detail）   │
│    │     │   每个 SKU 分配到哪个仓库，锁定多少              │
│    │     │                                                 │
│    │     ├─ 更新库存（wms_ware_sku）                       │
│    │     │   stock_locked += lock_quantity                 │
│    │     │   stock = stock_total - stock_locked            │
│    │     │                                                 │
│    │     ├─ 记录库存变动日志（wms_stock_log）              │
│    │     │   change_type = ORDER_LOCK                     │
│    │     │                                                 │
│    │     └─ 库存不足？                                     │
│    │         是 → 抛异常 → Seata 回滚全局事务              │
│    │         否 → task_status = 1（已锁定）                │
│    │                                                       │
│    ├─ 3. 发送延迟消息（30分钟超时检查）                     │
│    └─ 4. 记录订单操作日志                                  │
│                                                            │
└─ Seata 全局提交 ──────────────────────────────────────────┘
```

### 支付成功后扣减库存

```
mall-order 支付成功
  │
  ├─ 更新订单状态 → status=1（待发货）
  │
  └─ Feign 调用 mall-ware 扣减库存
       │
       ├─ 查 wms_ware_order_task WHERE order_sn=?
       │
       ├─ 更新 wms_ware_sku
       │   stock_locked -= lock_quantity
       │   stock_total -= lock_quantity
       │   stock = stock_total - stock_locked
       │
       ├─ 更新工作单 task_status = 2（已扣减）
       │
       └─ 记录日志 change_type = ORDER_DEDUCT
```

### 订单取消后释放库存

```
mall-order 取消/超时
  │
  └─ Feign 调用 mall-ware 释放库存
       │
       ├─ 查 wms_ware_order_task WHERE order_sn=?
       │   task_status == 1（已锁定）才能释放
       │
       ├─ 更新 wms_ware_sku
       │   stock_locked -= lock_quantity
       │   stock = stock_total - stock_locked
       │
       ├─ 更新工作单 task_status = 3（已释放）
       │
       └─ 记录日志 change_type = ORDER_UNLOCK
```

## 3. 采购补货流程

```
┌─ 库存预警 ──────────────────────────────────────────────┐
│                                                          │
│  定时任务每30分钟扫描 wms_ware_sku                        │
│  WHERE stock <= low_stock                                │
│                                                          │
│  发现库存不足 → 自动创建采购单                            │
│                                                          │
└──────────────────────────────────────────────────────────┘
            │
            ▼
┌─ 创建采购单 ────────────────────────────────────────────┐
│                                                          │
│  wms_purchase（status=0 新建）                           │
│  wms_purchase_detail（每个需要补货的SKU）                 │
│                                                          │
└──────────────────────────────────────────────────────────┘
            │
            ▼
┌─ 采购完成 → 入库 ──────────────────────────────────────┐
│                                                          │
│  更新 wms_purchase status=3（已入库）                    │
│  更新 wms_ware_sku                                       │
│    stock_total += quantity                               │
│    stock = stock_total - stock_locked                    │
│  记录日志 change_type = PURCHASE_IN                      │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

---

## 4. 库存分配策略

当订单包含多个SKU时，需要决定从哪个仓库发货：

```
策略1：单仓库优先（当前实现）
  找一个库存最充足的仓库，尽量从一个仓库发

策略2：最优拆分（可扩展）
  按仓库分组，每个仓库满足部分SKU，运费最优
```

## 6. 索引设计

| 表 | 索引 | 用途 |
|---|---|---|
| wms_ware_sku | uk_ware_sku | 查某个仓库某个SKU的库存 |
| wms_ware_sku | idx_sku_id | 查某SKU在所有仓库的总库存 |
| wms_stock_log | idx_ware_sku | 查某个仓库某SKU的变动历史 |
| wms_purchase | uk_purchase_sn | 采购单号唯一查询 |
| wms_purchase | idx_ware_id | 按仓库查采购单 |
| wms_purchase_detail | idx_purchase_id | 查采购单的商品列表 |
| wms_ware_order_task | idx_order_sn | 按订单号查工作单 |
| wms_ware_order_task_detail | idx_task_id | 查工作单详情 |
