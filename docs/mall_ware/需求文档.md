# mall_ware 仓储库 — 需求文档

## 1. 模块概述

`mall_ware` 是 Mall-HJ 项目的仓储服务数据库，服务于 `mall-ware` 微服务。

核心职责：
- 多仓库管理
- 商品库存管理（仓库+SKU维度）
- 库存锁定/解锁（下单锁定、支付扣减、取消释放）
- 库存变动日志
- 采购补货流程

技术关联：
- Seata 分布式事务（与 mall-order 联动）
- 库存预警触发采购

---

## 2. 功能需求

### 2.1 多仓库管理

| 功能 | 描述 |
|---|---|
| 仓库信息 | 名称、地址、联系电话 |
| 默认仓库 | 设置一个默认发货仓库 |
| 启用/禁用 | 禁用的仓库不参与发货 |

### 2.2 库存管理

```
库存三个维度：
  stock_total  = 总库存（物理库存）
  stock_locked = 已锁定库存（未支付订单占用）
  stock        = 可用库存 = stock_total - stock_locked

下单流程：
  创建订单 → stock_locked += 1（锁定）
  支付成功 → stock_locked -= 1, stock_total -= 1（扣减）
  取消订单 → stock_locked -= 1（释放）
```

### 2.3 库存工作单（Seata 分布式事务）

```
mall-order 下单
  │ Feign 调用 mall-ware
  ▼
mall-ware 创建库存工作单：
  ├─ wms_ware_order_task（工作单，关联订单号）
  └─ wms_ware_order_task_detail（每个SKU锁定多少）
  │
  ├─ 锁定成功 → Seata 提交 → 订单创建成功
  └─ 锁定失败（库存不足）→ Seata 回滚 → 订单创建失败
```

### 2.4 库存变动日志

| 变动类型 | 说明 |
|---|---|
| ORDER_LOCK | 下单锁定库存 |
| ORDER_UNLOCK | 订单取消释放库存 |
| ORDER_DEDUCT | 支付成功扣减库存 |
| PURCHASE_IN | 采购入库 |
| REFUND | 退款退回库存 |

### 2.5 采购补货

```
采购流程：
  新建采购单 → 分配商品和数量 → 采购完成 → 入库

  库存预警：
    当 stock <= low_stock 时触发预警
    → 自动创建采购单
```

## 3. 数据库表清单

| # | 表名 | 类型 | 说明 |
|---|---|---|---|
| 1 | wms_ware_info | 配置表 | 仓库信息 |
| 2 | wms_ware_sku | 业务表 | 商品库存（仓库+SKU维度） |
| 3 | wms_stock_log | 日志表 | 库存变动日志 |
| 4 | wms_purchase | 业务表 | 采购单 |
| 5 | wms_purchase_detail | 业务表 | 采购单详情 |
| 6 | wms_ware_order_task | 业务表 | 库存工作单（Seata关联） |
| 7 | wms_ware_order_task_detail | 业务表 | 库存工作单详情 |
