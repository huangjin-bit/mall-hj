# mall_seckill 秒杀库 — 需求文档

## 1. 模块概述

`mall_seckill` 是 Mall-HJ 项目的秒杀与促销服务数据库，服务于 `mall-seckill` 微服务。

核心职责：
- 秒杀场次管理（定时上下架）
- 秒杀商品管理（限购防刷）
- 高并发秒杀下单（Redis + RabbitMQ）
- 优惠券管理
- 促销活动（满减、阶梯价）

技术关联：
- Redis（库存预热、限购校验、防重复购买）
- RabbitMQ（异步下单，削峰填谷）

## 2. 功能需求

### 2.1 秒杀场次管理

| 功能 | 描述 |
|---|---|
| 创建场次 | 设置场次名称、开始/结束时间 |
| 定时上下架 | 定时任务扫描场次状态（未开始→进行中→已结束） |
| 关联商品 | 场次下添加秒杀商品，设置秒杀价、库存、限购数量 |

### 2.2 秒杀下单流程

```
用户点击"抢购"
  │
  ├─ 1. 校验：活动是否在进行中？
  ├─ 2. 校验：是否超过限购数量？（Redis 查已购记录）
  ├─ 3. 扣减库存：Redis DECR（原子操作，防超卖）
  │     ├── 库存 >= 0 → 继续
  │     └── 库存 < 0 → 已售罄
  ├─ 4. 发送消息到 RabbitMQ → 异步创建订单
  └─ 5. 返回"排队中"，前端轮询结果
```

### 2.3 限购防刷

| 措施 | 说明 |
|---|---|
| 每人限购 | 同一场次同一SKU，每人只能买N件 |
| Redis 记录 | seckill:bought:{sessionId}:{skuId}:{memberId} → 已购数量 |
| 接口限流 | 同一用户1秒内只能请求1次（Redis 令牌桶） |
| 隐藏地址 | 秒杀接口URL动态生成，防止提前知道直接刷 |

### 2.4 优惠券管理

| 功能 | 描述 |
|---|---|
| 创建优惠券 | 满减券、折扣券、无门槛券 |
| 领取 | 用户领取，检查库存和限领数量 |
| 使用 | 下单时使用，校验门槛和有效期 |
| 退还 | 订单取消时退回优惠券 |

### 2.5 促销活动

| 类型 | 说明 | 示例 |
|---|---|---|
| 满减 | 满 X 元减 Y 元 | 满200减30 |
| 阶梯价 | 买越多越便宜 | 买3件95折，买5件9折 |
