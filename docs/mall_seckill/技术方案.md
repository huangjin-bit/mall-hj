# mall_seckill 秒杀库 — 技术方案

## 1. 技术选型

| 项目 | 选型 |
|---|---|
| 数据库 | MySQL 8.0+, InnoDB, utf8mb4 |
| ID 生成 | 雪花算法（MyBatis-Plus ASSIGN_ID） |
| 库存预热 | Redis（活动开始前将库存加载到 Redis） |
| 异步下单 | RabbitMQ（削峰填谷） |
| 限购防刷 | Redis 原子操作 + 接口限流 |

---

## 2. 秒杀核心流程

```
┌─ 活动预热（开始前）───────────────────────────────────────┐
│                                                            │
│  定时任务扫描即将开始的场次                                  │
│  将该场次下所有商品的库存加载到 Redis：                      │
│                                                            │
│  seckill:stock:{sessionId}:{skuId}  →  100（库存数量）      │
│  seckill:limit:{sessionId}:{skuId}  →  1（限购数量）        │
│  seckill:price:{sessionId}:{skuId}  →  2999.00（秒杀价）    │
│                                                            │
│  设置过期时间 = 场次结束时间                                 │
│                                                            │
└────────────────────────────────────────────────────────────┘
            │
            ▼
┌─ 用户抢购（进行中）───────────────────────────────────────┐
│                                                            │
│  用户点击"立即抢购"                                         │
│    │                                                       │
│    ├─ 1. 校验活动状态                                       │
│    │     当前时间是否在 start_time ~ end_time 之间          │
│    │                                                       │
│    ├─ 2. 接口限流（1秒1次）                                 │
│    │     SET seckill:limit_req:{memberId} 1 EX 1 NX        │
│    │     已存在 → 拒绝（"操作太频繁"）                       │
│    │                                                       │
│    ├─ 3. 限购校验                                           │
│    │     GET seckill:bought:{sessionId}:{skuId}:{memberId}  │
│    │     >= limit → 拒绝（"已达到限购上限"）                 │
│    │                                                       │
│    ├─ 4. 扣减 Redis 库存（原子操作）                        │
│    │     DECR seckill:stock:{sessionId}:{skuId}             │
│    │     结果 < 0 → INCR 回滚 → 拒绝（"已售罄"）            │
│    │     结果 >= 0 → 继续                                   │
│    │                                                       │
│    ├─ 5. 记录已购数量                                       │
│    │     INCR seckill:bought:{sessionId}:{skuId}:{memberId} │
│    │     EXPIRE = 场次结束时间                               │
│    │                                                       │
│    ├─ 6. 发送消息到 RabbitMQ                                │
│    │     Exchange: seckill.order.exchange                   │
│    │     RoutingKey: seckill.order.create                   │
│    │     Body: {memberId, sessionId, skuId, seckillPrice}   │
│    │                                                       │
│    └─ 7. 返回"排队中"                                       │
│                                                            │
└────────────────────────────────────────────────────────────┘
            │
            ▼
┌─ 异步创建订单（RabbitMQ 消费者）──────────────────────────┐
│                                                            │
│  消费者收到消息：                                           │
│    │                                                       │
│    ├─ 1. 创建秒杀订单（sms_seckill_order, status=0）       │
│    ├─ 2. 远程调用 mall-order 创建正式订单（order_type=1）   │
│    │     → 关联 oms_order.id 到 sms_seckill_order.order_id │
│    ├─ 3. 远程调用 mall-ware 锁定库存                       │
│    └─ 4. 返回结果                                          │
│                                                            │
│  失败处理：                                                 │
│    INCR seckill:stock:{sessionId}:{skuId}（回滚Redis库存） │
│    DECR seckill:bought:{sessionId}:{skuId}:{memberId}      │
│                                                            │
└────────────────────────────────────────────────────────────┘
            │
            ▼
┌─ 前端轮询结果 ───────────────────────────────────────────┐
│                                                            │
│  GET /seckill/result?sessionId=1&skuId=100                  │
│                                                            │
│  查 sms_seckill_order 是否存在：                             │
│    不存在 → 返回"排队中"                                    │
│    存在 + status=0 → 返回"待支付" + orderId                 │
│    存在 + status=2 → 返回"已取消"                           │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

## 3. Redis Key 设计

```
# 库存（预热加载，活动结束自动过期）
seckill:stock:{sessionId}:{skuId}             → 100

# 限购数量（预热加载）
seckill:limit:{sessionId}:{skuId}             → 1

# 秒杀价（预热加载）
seckill:price:{sessionId}:{skuId}             → 2999.00

# 已购记录（抢购成功后写入）
seckill:bought:{sessionId}:{skuId}:{memberId} → 1

# 接口限流（每次请求写入，1秒过期）
seckill:limit_req:{memberId}                  → 1

# 秒杀结果缓存
seckill:result:{sessionId}:{skuId}:{memberId} → orderId / "waiting"
```

## 5. 索引设计

| 表 | 索引 | 用途 |
|---|---|---|
| sms_seckill_session | idx_start_time | 定时任务扫描即将开始的场次 |
| sms_seckill_sku_relation | idx_session_id | 查场次下的商品 |
| sms_seckill_order | uk_member_session_sku | 防重复抢购 |
| sms_seckill_order | idx_order_id | 查正式订单关联的秒杀订单 |
| sms_coupon_history | idx_coupon_id | 查优惠券领取记录 |
| sms_coupon_history | idx_member_id | 用户查自己的优惠券 |
