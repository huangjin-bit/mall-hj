# mall_order 订单库 — 技术方案

## 1. 技术选型

| 项目 | 选型 |
|---|---|
| 数据库 | MySQL 8.0+, InnoDB, utf8mb4 |
| ID 生成 | 雪花算法（MyBatis-Plus ASSIGN_ID） |
| 分布式事务 | Seata 2.6.0（AT 模式，order ↔ ware） |
| 消息队列 | RabbitMQ（延迟队列实现订单超时取消） |
| 购物车存储 | Redis（主） + MySQL（副本） |
| 支付对接 | 支付宝 SDK / 微信支付 SDK / 模拟支付 |

## 3. 下单流程（Seata 分布式事务）

```
用户点击"提交订单"
        │
        ▼
┌─ 开启 Seata 全局事务 ───────────────────────────────────────┐
│                                                               │
│  1. 创建订单（oms_order + oms_order_item）                    │
│     → 计算总金额、优惠、运费                                   │
│     → 保存收货地址快照                                         │
│                                                               │
│  2. 远程调用 mall-ware 扣库存（Feign）                         │
│     → wms_ware_sku: stock = stock - quantity                  │
│     → Seata 保证：库存扣了但订单创建失败 → 自动回滚库存        │
│                                                               │
│  3. 清除购物车中已勾选的商品                                   │
│     → 删除 Redis 中对应 key                                   │
│     → 删除 oms_cart_item 中对应记录                           │
│                                                               │
│  4. 发送延迟消息到 RabbitMQ（30分钟后检查）                    │
│     → 如果 30 分钟后订单仍为"待付款"，触发自动取消              │
│                                                               │
│  5. 记录操作日志（oms_order_operate_history）                  │
│                                                               │
└─ Seata 全局事务提交/回滚 ───────────────────────────────────┘
```

## 4. 订单超时自动取消（RabbitMQ 延迟队列）

```
下单成功
  │
  │ 发送消息到 order.delay.queue（TTL=30min，无消费者）
  ▼
┌──────────────────────┐
│  order.delay.queue   │  消息过期后通过 DLX 转发
│  TTL = 30 min        │──────────────────────┐
└──────────────────────┘                      │
                                              ▼
                                    ┌────────────────────┐
                                    │ order.close.queue  │
                                    │ 消费者：检查订单状态 │
                                    └────────┬───────────┘
                                             │
                                    ┌────────┴───────────┐
                                    │ status == 0(待付款)?│
                                    └───┬──────────┬─────┘
                                        │          │
                                      是           否
                                        │          │
                                        ▼          ▼
                                   取消订单     什么都不做
                                   释放库存
                                   退回优惠券
                                   记录操作日志
```

## 5. 支付流程

```
用户点击"去支付"
  │
  ▼
mall-order 创建支付记录（oms_payment_info, status=0）
  │
  ├─ 支付宝：调用支付宝 SDK 创建交易 → 返回支付页面 URL → 重定向
  ├─ 微信：调用微信支付 API → 返回二维码 URL → 前端展示
  └─ 模拟：直接标记为已支付
  │
  ▼
用户完成支付
  │
  ▼
第三方异步回调 mall-order /pay/callback
  │
  ├─ 验证签名（防伪造回调）
  ├─ 幂等处理（同一笔订单只处理一次）
  ├─ 更新 oms_payment_info（status=1, callback_content, callback_time）
  ├─ 更新 oms_order（status=1, pay_time, pay_type）
  └─ 记录操作日志
```

## 6. 购物车 Redis + 数据库双写

```
┌─ 写操作（加入/修改/删除）──────────────────┐
│                                             │
│  1. 写 Redis（主）                          │
│     HSET cart:{memberId} {skuId} {JSON}     │
│                                             │
│  2. 异步写数据库（副本）                     │
│     发送消息到 RabbitMQ → 消费者写入 DB      │
│                                             │
└─────────────────────────────────────────────┘

┌─ 读操作（查看购物车）───────────────────────┐
│                                             │
│  1. 读 Redis                                │
│     HGETALL cart:{memberId}                 │
│                                             │
│  2. Redis 为空 → 读数据库 → 回写 Redis      │
│                                             │
└─────────────────────────────────────────────┘

Redis 数据结构：
  Key:   cart:{memberId}
  Field: {skuId}
  Value: {"skuId":xxx,"spuId":xxx,"skuName":"...","skuImg":"...",
          "skuPrice":5999.00,"quantity":2,"isChecked":1}
```

## 8. 索引设计

| 表 | 索引 | 用途 |
|---|---|---|
| oms_order | uk_order_sn | 订单号唯一查询 |
| oms_order | idx_member_id | 用户查自己的订单 |
| oms_order | idx_status, idx_create_time | 管理后台按状态/时间筛选 |
| oms_order_item | idx_order_id | 查订单的商品列表 |
| oms_order_operate_history | idx_order_id | 查订单操作历史 |
| oms_payment_info | uk_order_sn | 按订单号查支付信息 |
| oms_payment_info | idx_pay_sn | 按第三方流水号查询 |
| oms_order_return_apply | idx_order_sn, idx_member_id | 退款申请查询 |
| oms_cart_item | uk_member_sku | 防重复加入购物车 |
