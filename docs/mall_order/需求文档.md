# mall_order 订单库 — 需求文档

## 1. 模块概述

`mall_order` 是 Mall-HJ 项目的订单服务数据库，服务于 `mall-order` 微服务。

核心职责：
- 购物车管理（Redis + 数据库双写）
- 订单创建、状态流转
- 订单超时自动取消（RabbitMQ 延迟队列）
- 支付对接（微信/支付宝/模拟）
- 退款退货申请与审核

技术关联：
- Seata 分布式事务（下单时同步扣库存，与 mall-ware 联动）
- RabbitMQ（订单超时取消、秒杀下单异步处理）

## 2. 功能需求

### 2.1 购物车

| 功能 | 描述 |
|---|---|
| 加入购物车 | 添加 SKU，已存在则数量+1 |
| 修改数量 | 修改某个 SKU 的购买数量 |
| 删除 | 删除单个或批量删除 |
| 勾选 | 勾选/取消勾选，结算时只计算已勾选项 |
| 双写策略 | Redis 为主存储（读写快），数据库做副本（防丢失） |

### 2.2 订单状态流转

```
[待付款] ──支付──→ [待发货] ──发货──→ [待收货] ──确认收货──→ [已完成]
   │                  │
   │超时30min         │申请退款
   ▼                  ▼
[已取消]          [退款中] ──退款成功──→ [已退款]
```

| 状态码 | 状态 | 触发条件 |
|---|---|---|
| 0 | 待付款 | 下单成功 |
| 1 | 待发货 | 支付成功 |
| 2 | 待收货 | 商家发货 |
| 3 | 已完成 | 用户确认收货 |
| 4 | 已取消 | 超时未支付 / 用户手动取消 |
| 5 | 退款中 | 用户申请退款 |
| 6 | 已退款 | 退款成功 |
| 7 | 售后中 | 已完成订单申请售后 |
| 8 | 已关闭 | 售后完成 |

### 2.3 订单超时自动取消

- 下单后 30 分钟未支付 → 自动取消
- 实现：RabbitMQ 延迟队列（死信交换机）
- 取消时：释放库存、退回优惠券、记录操作日志

### 2.4 支付

| 支付方式 | 说明 |
|---|---|
| 支付宝 | 对接支付宝沙箱/正式环境 |
| 微信支付 | 对接微信支付 API |
| 模拟支付 | 开发阶段用，直接标记为已支付 |

支付流程：
1. 创建支付记录（oms_payment_info，status=0）
2. 调用第三方支付 API，返回支付页面/二维码
3. 用户完成支付，第三方异步回调通知
4. 验证回调签名，更新支付状态，触发订单状态变更

### 2.5 退款退货

| 功能 | 描述 |
|---|---|
| 仅退款 | 未发货/已发货未收货，申请退钱 |
| 退货退款 | 已收货，退回商品并退款 |
| 审核流程 | 管理员审核同意/拒绝 |
| 退款原因 | 管理员配置退款原因选项 |

## 4. 关键设计说明

### 4.1 订单项数据快照

订单项（oms_order_item）冗余存储了商品名称、SKU名称、单价、图片等，这是下单时的快照。商品表后续修改不影响历史订单展示。

### 4.2 金额计算

```
实付金额 = 商品总价 + 运费 - 促销优惠 - 优惠券抵扣
pay_amount = total_amount + freight_amount - promotion_amount - coupon_amount
```

促销优惠和优惠券按比例分摊到每个订单项，方便部分退款时计算。

### 4.3 订单编号规则

格式：yyyyMMddHHmmss + 6位随机数，如 `20260504143025001234`
