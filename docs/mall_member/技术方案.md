# mall_member 会员库 — 技术方案

## 1. 技术选型

| 项目 | 选型 |
|---|---|
| 数据库 | MySQL 8.0+, InnoDB, utf8mb4 |
| ID 生成 | 雪花算法（MyBatis-Plus ASSIGN_ID） |
| 密码加密 | BCrypt（自带盐值） |
| Token 方案 | JWT + Redis |
| OAuth2 框架 | 自实现（授权码模式） |
| 连接池 | HikariCP（Spring Boot 默认） |

## 3. 安全设计

### 3.1 授权请求阶段

| 威胁 | 防护手段 |
|---|---|
| CSRF 攻击 | `state` 参数随机生成，回调时严格比对 |
| 授权码截获 | PKCE（code_challenge + code_verifier） |
| 恶意重定向 | `redirect_uri` 严格全匹配，不允许通配符 |
| scope 越权 | 检查请求 scope 是否在 sys_client 允许范围内 |

### 3.2 授权码阶段

| 威胁 | 防护手段 |
|---|---|
| 授权码窃取重放 | Redis 存储一次性使用，5分钟过期，用完即删 |
| 暴力猜解 code | code 使用 SecureRandom，至少32字节 |

### 3.3 Token 阶段

| 威胁 | 防护手段 |
|---|---|
| Token 伪造 | JWT 使用 RS256 非对称签名，私钥只在 auth 服务 |
| Token 过期后使用 | access_token 2小时过期 |
| Token 被盗 | Redis 存副本，支持即时撤销 |
| Refresh Token 被盗 | 刷新时轮换，旧 token 立即失效 |

### 3.4 密码安全

| 措施 | 说明 |
|---|---|
| BCrypt 加密 | 自带盐值，cost factor = 10 |
| 密码强度校验 | 最少8位，含大小写+数字 |
| 登录失败限制 | 同一账号5次失败锁定30分钟（Redis 计数） |

### 3.5 主动让 Token 失效

| 场景 | 操作 |
|---|---|
| 用户改密码 | 删除 Redis 中该用户所有 token |
| 用户被禁用 | 同上 |
| 管理员踢人 | 删除指定 token |
| 用户主动登出 | 删除当前 token |

## 4. Token 存储策略

### Redis Key 设计

```
# 授权码（5分钟，一次性）
oauth2:code:{code}                    → {memberId, clientId, scope, codeChallenge, redirectUri}

# access_token 副本（2小时）
oauth2:access:{memberId}:{jti}        → tokenValue

# refresh_token（30天）
oauth2:refresh:{refreshToken}         → {memberId, clientId, scope}

# 用户授权同意缓存（7天，加速查询）
oauth2:consent:{memberId}:{clientId}  → scopes

# 登录失败计数（30分钟后重置）
auth:fail:{loginId}                   → count
```

## 6. 索引设计说明

| 表 | 索引 | 用途 |
|---|---|---|
| ums_member | uk_username, uk_phone | 登录查询、防重复 |
